# ClearCue 🎙️

> **ClearCue** is an interactive, AI-powered interview practice application designed to help candidates prepare for their next big career move with confidence. Get real-time feedback, practice with tailored questions, and refine your pitch!

## ✨ Key Features

- **Personalized Interview Prep**: Tailored questions generated by Groq AI (Llama 3.3 70B) based on your target job title, industry, and experience level.
- **Voice-Based Interviews**: Browser-native speech recognition captures your answers in real-time, with text-to-speech reading questions aloud. Manual text input available as fallback.
- **Instant Performance Analysis**: Every answer is scored across 5 dimensions — Clarity, Confidence, Structure, Specificity, and Impact — with actionable strengths and improvement tips.
- **Performance Dashboard**: Radar charts, weekly trend bars, and downloadable PNG scorecards to track your progress over time.
- **Gamification**: Practice streaks, milestone badges (1/3/7/30 days), and progress tracking to keep you motivated.
- **Secure Authentication**: NextAuth v5 with JWT sessions, bcrypt password hashing, email validation, and route protection via Edge middleware.
- **Rate Limiting**: Built-in IP-based rate limiting on login (5 req/min) and registration (3 req/min) to prevent abuse.

## 🚀 Getting Started

### Prerequisites

- Node.js v18+
- npm, yarn, or pnpm
- MongoDB Atlas account (or local MongoDB instance)
- Groq API key (for AI question generation)

### Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/your-username/ClearCue.git
   cd ClearCue
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Set up environment variables:**

   Create a `.env.local` file in the root directory:
   ```env
   # Database
   MONGODB_URI=mongodb+srv://<user>:<password>@cluster.mongodb.net/clearcue

   # NextAuth
   AUTH_SECRET=your_random_32_character_secret_here

   # Groq API (for AI question generation)
   GROQ_API_KEY=your_groq_api_key
   ```

4. **Run the development server:**
   ```bash
   npm run dev
   ```

5. Open [http://localhost:3000](http://localhost:3000) in your browser.

## 📊 Scoring System

Each answer is analyzed across 5 dimensions:

| Dimension | What It Measures |
|-----------|-----------------|
| **Clarity** | Answer length and conciseness (40-220 words optimal) |
| **Confidence** | Ownership language and sufficient depth |
| **Structure** | Use of frameworks (first, second, finally, etc.) |
| **Specificity** | Concrete examples and measurable metrics |
| **Impact** | Results-oriented language and business outcomes |

Additional factors: filler word detection, speaking pace (WPM), and penalization for overly verbose or empty answers.

## 🔒 Security

- Passwords hashed with bcrypt (cost factor 12)
- JWT-based sessions with dev-mode invalidation on restart
- Email format validation on client and server
- IP-based rate limiting on auth endpoints
- Edge middleware protects all authenticated routes
- Internal error messages never exposed to clients
